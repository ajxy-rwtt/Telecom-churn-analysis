# 📊 Telecom Churn Analysis

## 📌 Overview

Analyzed telecom customer data to identify key churn drivers, segment high-risk users, and quantify revenue impact using SQL and Power BI.

---

## 🛠️ Work Done

* Cleaned and structured raw dataset using **SQL**
* Engineered key features:

  * `total_usage` (combined minutes)
  * `total_bill` (total revenue per customer)
* Segmented customers based on:

  * Tenure (`account_length`)
  * Complaint frequency (`customer_service_calls`)
* Built a **rule-based churn risk model**
* Developed an interactive **Power BI dashboard**

---

## 📊 Key Insights

* 📞 **Customer complaints are the strongest churn driver** (risk spikes after 3+ calls)
* 💰 **Revenue risk exists** — high-value customers are also churning
* 🌍 **Churn varies by state**, indicating regional issues
* 📉 **New customers churn more**, highlighting onboarding gaps
* 🌐 **International plan users (~10%)** form a distinct behavioral segment

---

## 📈 Dashboard Highlights

* KPIs: Revenue, Churn Distribution, Avg Usage
* Churn analysis by:

  * State
  * Service calls
  * Plan type
* Risk segmentation (High / Medium / Low)

---

## ⚙️ Tools

* SQL (MySQL)
* Power BI
* Python (EDA)

---

## 🎯 Outcome

Delivered actionable insights to help reduce churn through targeted retention strategies and improved customer experience.
